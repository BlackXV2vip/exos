# 🌐 بلاغ مستخدم + إصلاح — Web UI "زرار الجلسة مش بيستجيب" (v1.18.7)

**البلاغ (من تليفون المستخدم):** «حاولت اعمل جلسة ماعملش حاجة، لذلك لم أتمكن من إرسال الرسائل».

## إعادة الإنتاج (متصفح headless حقيقي + مراقبة كل الطلبات)

أعدنا إنتاج المشكلة بدقة بـ Playwright + Chromium ضد باينري v1.18.6 المنشور:

| الخطوة | النتيجة |
|---|---|
| فتح `http://127.0.0.1:8799/` | ✅ الواجهة تفتح (0 أخطاء كونسول) وكل طلبات التهيئة 200 |
| ضغط زرار «New session» | 🔴 **ولا حاجة**: لا تنقّل، لا طلب API، صمت تام |

## التحليل الجذري (Root Cause)

السبب مش في السيرفر — `POST /session` كان شغال 100% عبر curl. السبب في الواجهة نفسها: زرار «New session» بيعمل **رجوع صامت** (`if (!fallback) return`) لما قائمة المشاريع المحلية تكون فاضية — وهي بتتعبّى يدويًا بزرار «Add project». على سيرفر جديد في مجلد من غير git (زي `$HOME` على الموبايل)، مفيش مشروع جيت حقيقي (السيرفر بيرجع placeholder اسمه `global`) فالقائمة بتفضل فاضية → الزرار ميت. المسار الشغال في upstream بيعتمد إن أول مجلد يكون git repo — وده مششرط منطقي على الإطلاق.

## الإصلاح (ملفان، باتش جراحي)

1. `packages/app/src/pages/home.tsx` — لو مفيش مشاريع مضافة، «New session» بيفولباك لمسار عمل السيرفر الحالي (`/path → directory`) بدل ما يبقى معطّل + الـ CTA المركزي بيظهر بدل ما يختفي.
2. `packages/app/src/components/titlebar.tsx` — زرار الأيقونة (باختصار Ctrl+T): آخر درجة فولباك بقت طلب `/path` من السيرفر واستخدام مساره الحالي بدل الرجوع الصامت.
